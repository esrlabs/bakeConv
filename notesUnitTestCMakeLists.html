Notes for CMakeLists for UnitTest

If a separate CMakeLists.txt for UnitTest should be generate, some points have to take into account:
  • In the config file one has to define
    • GMOCK_FILTER = true

    This has the consequence, that the gmock pathes will be removed. "make" will do not have problems if gmock is filtered, otherwise linker errors will be occur.

  • Additionally one has to define
    • DEPENDENCIES_FILTER = true/false

    If DEPENDENCIES_FILTER is set "true", $$(DEPENDENCIES_FILTER) in the template file will be replaced by the content of BAKE_DEPENDENCIES_FILTERED respectively the content of BAKE_DEPENDENCIES if the value is "false".

    Example: (DEPENDENCIES_FILTER = true)

    template_UnitTest.txt CMakeLists.txt 
     ...
    
 target_link_libraries(gtest_${targetName}
 gtest_${targetName}_sut
 gtest_esr_all_$$(DEPENDENCIES_FILTER)_sut
 ${testFrameworkTarget}
 )
	
     ...
    
 target_link_libraries(gtest_${targetName}
 gtest_${targetName}_sut
 gtest_esr_all_logger_sut
 ${testFrameworkTarget}
 )
